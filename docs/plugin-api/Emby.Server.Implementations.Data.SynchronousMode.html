    Enum SynchronousMode

    The disk synchronization mode, controls how aggressively SQLite will write data all the way out to physical storage.

    Namespace: Emby.Server.Implementations.Data
    Assembly: Emby.Server.Implementations.dll
    Syntax
    public enum SynchronousMode

    Fields

    Name Description
    Extra

    EXTRA synchronous is like FULL with the addition that the directory containing a rollback journal is synced after that journal is unlinked to commit a transaction in DELETE mode.
    Full

    SQLite database engine will use the xSync method of the VFS to ensure that all content is safely written to the disk surface prior to continuing.
    Normal

    SQLite database engine will still sync at the most critical moments
    Off

    SQLite continues without syncing as soon as it has handed data off to the operating system
